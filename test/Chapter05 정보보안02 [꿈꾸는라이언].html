Chapter05 정보보안02 [꿈꾸는 라이언]

  • 침입 탐지 시스템 (IDS, Intrusion Detection System)
  • 시스템의 비정상적인 사용, 오용, 남용 등을 실시간으로 탐지하여 알리는 보안 시스템입니다.
  • 오용 탐지 (Misuse Detection)
  • 사전에 정립되어 입력해 둔 공격 패턴을 감지할 경우 이를 통보하는 침입 탐지 시스템의 방식입니다.
  • 이상 탐지 (Anomaly Detection)
  • 평균적인 상태 기준에서 벗어난 비정상적인 행위 또는 자원의 사용을 감지할 경우 통보하는 침입 탐지 시스템의 방식입니다.
  • HIDS (Host-based Intrusion Detection System)
  • 시스템 내부를 감시/분석하며, 내부 변화를 실시간으로 감시하여 외부 침입자의 작업 기록 및 추적에 사용되는 IDS 종류입니다 (ex. OSSEC, AIDE).
  • NIDS (Network-based Intrusion Detection System)
  • 외부 침입을 감시/분석하며, 네트워크 트래픽을 감시하여 침입을 탐지하는 IDS 종류입니다 (ex. Snort, Zeek).
  • SSH (Secure SHell)
  • 서로 연결된 컴퓨터 간 원격 명령 실행 및 셀 서비스를 수행하며, 전송 데이터를 암호화하고 기본적으로 22번 포트를 사용하는 보안 프로토콜입니다.
  • SSL (Secure Socket Layer)
  • 웹 브라우저 서버 간 안전한 데이터 전송을 목적으로 하며, https로 시작하고 443번 포트를 사용하는 보안 프로토콜입니다.
  • TLS (Transport Layer Security)
  • 인터넷 커뮤니케이션을 위한 개인 정보와 데이터 무결성을 제공하는 보안 프로토콜로, SSL의 개선 버전입니다.
  • IPSec (Internet Protocol Security)
  • IP 네트워크 계층을 안전하게 보호하기 위한 보안 프로토콜로, **AH(Authentication Header)**와 **ESP(Encapsulating Security Payload)**라는 두 가지 보안 프로토콜을 사용합니다.
  • TKIP (Temporal Key Integrity Protocol)
  • WEP 취약성을 보완하기 위해 암호 알고리즘의 입력 키 길이를 128비트로 늘린 임시 키 무결성 프로토콜입니다.
  • 생략 오류 (Omission error)
  • 데이터 입력 시 한 자리를 빼놓고 기록하는 오류입니다 (ex. ABCD → ABD).
  • 필사 오류 (Transcription error)
  • 데이터 입력 시 임의의 한 자리를 잘못 기록하는 오류입니다 (ex. ABCD → ABFD).
  • 전위 오류 (Transposition error)
  • 데이터 입력 시 좌우 자리를 바꾸어 기록하는 오류입니다 (ex. ABCD → ACBD).
  • ISMS (Information Security Management System)
  • 정보보호 관리 체계로, 정보 자산을 안전하게 보호하기 위한 보호 절차와 대책을 확보하고 조직에 맞는 정보보호 정책을 수립하는 활동입니다.
  • 다크 데이터 (Dark Data)
  • 특정 목적으로 수집되었으나 활용되지 않고 저장만 되어 있는 대량의 데이터입니다.
  • 스캐어웨어 (Scareware)
  • 가짜 바이러스 경고나 시스템 문제를 표시하여 사용자를 속여 돈을 지불하거나 특정 소프트웨어를 설치하도록 유도하는 악성 프로그램입니다.
  • 해킹 (Hacking)
  • 시스템에 침입하여 정보를 수정하거나 빼내는 행위입니다.
  • 크래킹 (Cracking)
  • 시스템에 침입하여 정보를 파괴하거나 변경하는 행위입니다.
  • 좀비 PC (Zombie PC)
  • 악성코드에 감염되어 해커에게 조종당하며 다른 컴퓨터를 공격하는 데 사용되는 PC입니다.
  • C&C 서버 (Command & Control Server)
  • 해커가 감염된 좀비 PC에 명령을 내리고 악성코드를 제어하기 위한 용도로 사용하는 서버입니다.
  • 웜 (Worm)
  • 다른 컴퓨터의 취약점을 이용하여 스스로 전파하거나 메일로 전파되며, 독자적으로 실행 가능한 악성 소프트웨어입니다.
  • 바이러스 (Virus)
  • 파일이나 메모리 영역에 **자신을 복제(기생)**하는 악성 프로그램으로, 독자적으로 실행은 불가능합니다.
  • 트로이목마 (Trojan Horse)
  • 정상적인 프로그램으로 가장하여 숨어 있으며, 사용자 실행 시 악성 기능을 수행하지만 스스로 복제하지는 않는 악성 프로그램입니다.
  • 백도어 (Backdoor)
  • 보안이 제거된 비밀 통로로, 무단 접근을 위해 시스템에 만들어 둔 통로입니다. (탐지 방법: 무결성 검사, 로그 분석 등)
  • 랜섬웨어 (Ransomware)
  • 내부 문서 파일 등을 암호화해 사용자가 열지 못하게 하고 이를 인질로 금전을 요구하는 악성 프로그램입니다.
  • APT (Advanced Persistent Threat)
  • 지능형 지속 공격으로, 특정 개인 및 조직을 타겟으로 침투 후 장기간에 걸쳐 내부 서버 제어권을 획득하여 데이터를 수집하는 공격입니다.
  • Qshing (큐싱)
  • QR 코드를 통해 악성 앱 다운을 유도하거나 설치하도록 하는 공격입니다.
  • XSS (Cross Site Scripting)
  • 브라우저 스크립트 취약점을 악용하여 특정 링크 클릭 시 악성 스크립트가 실행되어 개인 정보를 탈취하는 공격입니다.
  • CSRF (Cross-Site Request Forgery)
  • 사용자가 공격자가 의도한 행위 (수정, 삭제, 등록 등)를 특정 웹사이트에 요청하게 하는 공격입니다.
  • 제로 데이 공격 (Zero-Day Attack)
  • 발견된 취약점의 존재를 공표하기 전에 해당 취약점을 이용하여 이루어지는 보안 공격입니다.
  • 스니핑 (Sniffing)
  • 네트워크를 통해 전송되는 패킷을 엿보면서 계정 정보(ID/PW)를 가로채는 행위입니다.
  • 스푸핑 (Spoofing)
  • 검증된 사람이 네트워크를 통해 데이터를 보낸 것처럼 데이터를 변조하여 위장 접속을 시도하는 일종의 속임수입니다. (IP, ARP, DNS Spoofing 등)
  • 세션 하이재킹 (Session Hijacking)
  • 서버에 접속하고 있는 클라이언트들의 세션 정보를 가로채 서버와 클라이언트 사이의 연결을 가로채는 공격입니다.
  • 스피어 피싱 (Spear Phishing)
  • 조직 내 특정 개인 또는 그룹을 대상으로, 신뢰할 수 있는 발신자가 보낸 것처럼 속여 금융 개인 정보를 탈취하는 피싱 공격입니다.
  • 스미싱 (Smishing)
  • **문자 메시지(SMS)**를 통해 사용자를 속여 개인 신용 정보를 탈취하는 공격입니다.
  • 파밍 (Pharming)
  • 금융기관의 도메인 주소(DNS)를 중간에 가로채 사용자가 금융기관 사이트에 접속한 듯 착각하게 하여 개인 정보를 탈취하는 공격입니다.
  • 타이포스쿼팅 (Typosquatting)
  • 사용자가 웹 URL 주소를 잘못 입력하는 실수를 악용하여, 유사한 주소를 가진 악성 사이트로 유도하는 공격입니다.
  • 웨일링 (Whaling)
  • CEO, 고위 경영진, 연예인 등 유명인사를 타깃으로 한 스피어 피싱 공격입니다.
  • 키로거 (Keylogger)
  • 사용자의 키 입력을 캐치하는 프로그램을 이용하여 개인 정보를 빼내는 행위입니다.
  • 버퍼 오버플로우 (Buffer Overflow)
  • 프로그램에서 할당된 메모리의 범위를 넘어선 위치에서 자료를 사용하려고 할 때 발생하는 오류입니다.
  • SQL 삽입 공격 (SQL Injection Attack)
  • 임의의 SQL 문을 주입하고 실행되게 하여 데이터베이스가 비정상적인 동작을 하도록 조작하는 코드 삽입 공격입니다.
  • Brute Force Attack (무작위 대입 공격)
  • 가능한 모든 문자열 조합을 시도하여 암호키나 로그인 정보를 탈취하는 공격입니다.
  • Watering Hole Attack (워터링 홀 공격)
  • 타겟이 주로 방문하는 웹 사이트를 미리 감염시킨 후, 해당 타겟이 방문할 때까지 기다리는 공격입니다.
  • Bluebug (블루버그)
  • 블루투스 연결 취약점을 이용하여 휴대폰 등의 장치에 무단으로 접근하여 정보를 탈취하는 공격입니다.
  • Credential Stuffing
  • 공격자가 수집한 **로그인 인증 정보 (Credential)**를 다른 사이트의 계정 정보에 무작위로 대입하여 접속을 시도하는 공격입니다.
  • Island Hopping
  • 타겟 기업을 침해하기 위해 보안이 더 취약한 협력사, 파트너 기업의 네트워크를 해킹한 후 이를 경유하여 공격하는 기법입니다.
  • 사회 공학 (Social Engineering)
  • 인간 상호 작용의 깊은 신뢰를 바탕으로 사람들을 속여 보안 절차를 깨뜨리기 위한 비기술적 침입 수단입니다.
  • Rootkit (루트킷)
  • 권한이 없는 사용자가 접근할 수 없는 영역에 접근하여 시스템을 제어하도록 설계된 악성 소프트웨어의 모음으로, 해킹에 사용되는 기능들을 제공하는 프로그램들의 모음입니다.
  • S-HTTP (Secure Hypertext Transfer Protocol)
  • 웹상에서 네트워크 트래픽을 암호화하는 주요 방법 중 하나로, 기존 HTTP의 보안을 확장하여 클라이언트와 서버 간 전송되는 모든 메시지를 각각 암호화해 전송하는 기술입니다.
