Chapter01 소프트웨어 유지보수 [꿈꾸는 라이언]

  • 성능 분석 지표
  • 애플리케이션의 성능 개선을 위해 측정하는 지표들로, 처리량, 응답 시간, 경과 시간, 자원 사용률 등이 있습니다.
  • 코드 스멜 (Code Smell)
  • 소스 코드 내에 존재하는 잠재적인 문제점을 의미하며, 중복 코드, 긴 메서드, 큰 클래스, 클래스 동시 수정 등이 예시입니다. 코드를 최적화하여 이를 제거하면 품질이 개선됩니다.
  • 스파게티 코드 (Spaghetti Code)
  • 소스코드의 로직이 복잡하게 얽혀 있어 흐름을 파악하기 어렵고 유지보수가 어려운 구조의 코드입니다.
  • 외계인 코드 (Alien Code)
  • 개발된 지 오래되어 개발자나 현 시스템에 대한 이해가 부족하여 유지보수 작업이 어려운 코드입니다.
  • 정적 분석 도구 (Static Analysis Tool)
  • 프로그램을 실행하지 않고 코딩 표준, 스타일, 결함 등을 분석하여 소스코드 품질을 점검하는 도구입니다 (ex. PMD, Checkstyle, SonarQube).
  • 동적 분석 도구 (Dynamic Analysis Tool)
  • 프로그램을 실행하면서 코드 내의 메모리 누수 및 스레드 결함 등을 발견하는 소스코드 품질 분석 도구입니다 (ex. Avalanche, Valgrind).
  • 소프트웨어 형상 관리 (SCM, Software Configuration Management)
  • 소프트웨어 개발 과정에서 발생하는 변경 사항을 관리하기 위한 일련의 활동으로, 변경 사항 추적/통제, 무절제한 변경 방지, 개발 진행 이력 확인 등의 역할을 수행합니다.
  • 형상 식별 (Configuration Identification)
  • 관리 대상에 이름/번호를 부여한 후 계층 구조로 구분하여 수정 및 추적이 용이하도록 하는 형상 관리 활동입니다.
  • 형상 통제 (Configuration Control)
  • 식별된 형상 항목에 대한 변경 요구를 검토하여 기준선 (Base line)에 반영될 수 있도록 관리하는 형상 관리 활동입니다.
  • 형상 감사 (Configuration Audit)
  • 기준선 (Base line)의 무결성을 평가하기 위해 확인, 검증, 검열 과정을 진행하는 형상 관리 활동입니다.
  • 형상 기록 (Configuration Status Accounting)
  • 형상 식별, 통제, 감사 작업 결과를 기록/관리하고 보고서를 작성하는 형상 관리 활동입니다.
  • CVS (Concurrent Versions System)
  • 서버/클라이언트 구성으로, 다수의 인원이 동시에 버전 관리가 가능하도록 하는 클라이언트/서버형 버전 관리 도구입니다.
  • SVN (Subversion)
  • CVS를 개선한 툴로, 모든 개발은 trunk 디렉터리에서 수행되며, Commit 수행 시 revision이 1씩 증가하는 클라이언트/서버형 버전 관리 도구입니다.
  • Git
  • 서버 (원격) 저장소와 개발자 (지역) 저장소가 독립적인 분산된 P2P 모델 방식의 버전 관리 도구로, Commit 실수 발생 시에도 서버에 영향을 주지 않습니다.
  • ISO/IEC 12207 (기본 생명 주기 프로세스)
  • 국제 표준 소프트웨어 생명 주기로, 획득, 공급, 개발, 운영, 유지보수 프로세스로 구성됩니다.
  • CMMI 레벨 1 (초기, Initial)
  • 프로세스가 정의되어 있지 않아 작업자 개인의 능력에 따라 성공 여부가 결정되는, CMMI의 가장 낮은 성숙도 레벨입니다.
  • CMMI 레벨 2 (관리, Managed)
  • 규칙화 프로세스가 존재하며, 특정한 프로젝트 내에서 프로세스가 정의되고 수행되는 성숙도 레벨입니다.
  • CMMI 레벨 3 (정의, Defined)
  • 표준화 프로세스가 존재하며, 조직의 표준 프로세스를 활용하여 업무를 수행하는 성숙도 레벨입니다.
  • CMMI 레벨 4 (정량적 관리, Quantitatively Managed)
  • 예측 가능 프로세스가 존재하며, 프로젝트를 정량적으로 관리하고 통제하는 성숙도 레벨입니다.
  • CMMI 레벨 5 (최적화, Optimizing)
  • 지속 개선 프로세스가 존재하며, 프로세스 역량 향상을 위해 지속적인 개선 활동을 수행하는 CMMI의 가장 높은 성숙도 레벨입니다.
