SPICE (ISO 15504) 프로세스 능력 수준

  • SPICE (ISO 15504)
  • 소프트웨어 처리 개선 및 능력 평가 기준으로, 프로세스 성숙도를 6단계로 나누어 소프트웨어 조직의 프로세스 능력을 평가하는 모델입니다.
  • SPICE 수준 1 (불완전)
  • 프로세스가 구현되지 않거나 목적 달성이 불가한 상태의 프로세스 능력 수준입니다.
  • SPICE 수준 2 (수행)
  • 프로세스의 목적은 달성하지만, 그 수행이 계획되거나 축적되지 않은 상태의 프로세스 능력 수준입니다.
  • SPICE 수준 3 (관리)
  • 프로세스 수행이 계획되고 관리되는 상태의 프로세스 능력 수준입니다.
  • SPICE 수준 4 (확립)
  • 표준 프로세스를 이용하여 프로세스 수행이 계획되고 관리되는 상태의 프로세스 능력 수준입니다.
  • SPICE 수준 5 (예측 가능)
  • 표준 프로세스의 능력에 대해 정량적인 이해를 바탕으로 성능을 예측할 수 있는 상태의 프로세스 능력 수준입니다.
  • SPICE 수준 6 (최적)
  • 정의된 프로세스와 표준 프로세스가 지속적으로 개선되는 상태의 프로세스 능력 수준입니다.

URL 구성 요소

  • 스킴 (Scheme)
  • 사용할 프로토콜을 명시하는 URL 구성 요소입니다 (ex. http, https, ftp).
  • 호스트 (Host)
  • 도메인명 또는 IP 주소를 나타내는 URL 구성 요소입니다.
  • 포트 (Port)
  • Socket (소켓)을 구분하는 역할을 하며, 생략 시 http는 80, https는 443으로 기본값 설정되는 URL 구성 요소입니다.
  • 경로 (Path)
  • 호스트에서 제공하는 자원의 경로를 나타내는 URL 구성 요소입니다 (계층적 구조).
  • 질의 (Query)
  • Query parameter 또는 Query string으로 불리며, key=value 형태로 데이터를 전달하는 URL 구성 요소입니다 (ex. ?directoryNo=0&currentPage=1).
  • 프레그먼트 (Fragment)
  • 웹 문서 내부 북마크 기능 (해시태그)을 하며, 스크롤 이동 없이 해당 위치로 바로 이동 가능하게 하는 URL 구성 요소입니다.
  • Stack guard (스택 가드)
  • Stack 상 일정 주소 번지에 프로그래머가 유도하는 Canary를 심고, 스택이 붕괴/변조된 경우 Canary 체크 후 프로그램 실행을 비정상적으로 중단시키는 보안 기법입니다.
  • Docker (도커)
  • 컨테이너 기술을 자동화하여 쉽게 사용하는 오픈소스 프로젝트로, 소프트웨어 컨테이너 안에 응용 프로그램 배치 자동화 역할을 합니다.
  • Cipher Container (사이퍼 컨테이너)
  • JAVA에서 암호화/복호화 기능을 제공하는 컨테이너입니다.
  • Scytale (스키테일)
  • 암호화 기법 중 문자열의 위치를 바꾸는 방법(전치 암호)을 사용하는 방식입니다.
  • Tensor Flow (텐서플로우)
  • 2015년 공개된 구글 브레인 팀의 기계 학습 오픈소스 SW Library입니다.
  • One Seg (원 세그)
  • 일본/브라질에 상용 중인 디지털 TV 방송 기술입니다.
  • Foursquare (포스퀘어)
  • 위치 기반 소셜 네트워크 서비스입니다.
  • PaaS-Ta (파스-타)
  • 국내 IT 서비스 경쟁력 강화 목적으로 개발된 개방형 클라우드로, 인프라 제어, 관리/실행/개발/서비스/운영 환경으로 구성됩니다.
  • VLAN (Virtual LAN)
  • 물리적 배치와 상관없이 논리적으로 LAN을 구성하는 기술로, 접속된 장비들의 성능 향상 및 보안성 증대 효과가 있습니다.
  • SSO (Single Sign On)
  • 한 번의 로그인으로 다른 사이트 로그인도 허용하는 시스템입니다.
  • MQTT (Message Queuing Telemetry Transport)
  • TCP/IP 기반 네트워크에서 발행-구독 기반의 메시징 프로토콜로, 푸시 기술 기반의 경량 메시지 전송 프로토콜입니다.
  • Salt (솔트)
  • 동일한 패스워드들을 다른 암호 값으로 저장되도록 덧붙이는 무작위의 값입니다.
  • N-screen (앤-스크린)
  • N개의 서로 다른 단말기에서 동일 콘텐츠를 자유롭게 이용하는 서비스입니다.
  • ASLR (Address Space Layout Randomization)
  • 프로그램 실행마다 스택/힙/라이브러리 주소를 랜덤화하여 공격자로 하여금 메모리 상 주소 예측을 어렵게 하는 보안 기법입니다.
  • 라우터 (Router)
  • 서로 다른 네트워크 대역에 있는 호스트를 상호 간에 통신할 수 있도록 경로를 설정해 주는 네트워크 장비입니다.
  • nmap (Network Mapper)
  • 서버에 열린 포트 정보를 스캐닝하여 보안 취약점을 찾는 도구입니다.
  • Tripwire (트립와이어)
  • 크래커가 침입하여 백도어를 만들거나 설정 파일을 변경했을 때 이를 분석하는 도구입니다.
  • Smart Grid (스마트 그리드)
  • 정보 기술을 활용하여 전력망을 지능화, 고도화함으로써 고품질의 전력 서비스를 제공하고 에너지 이용 효율을 극대화하는 시스템입니다.
  • 서비스 지향 아키텍처 (SOA, Service Oriented Architecture)
  • 표현, 업무 프로세스, 서비스 중간, 애플리케이션, 데이터 저장 등의 계층으로 구성되는 아키텍처입니다.
  • Digital twin (디지털 트윈)
  • 물리적 사물을 가상화하여 실제 자산의 특성 정보를 구현하고, 자산 최적화 및 생산성 증가를 도모하는 기술입니다.
  • Mashup (매시업)
  • 웹에서 제공하는 정보/서비스를 이용하여 새로운 SW를 제작하는 기법입니다.
  • Mesh Network (메시 네트워크)
  • 특수 목적을 위한 네트워크 기술로, 대규모 네트워크 생성에 최적화되어 노드들이 그물망처럼 연결되는 방식입니다.
  • PICONET (피코넷)
  • 여러 개의 독립된 통신장치가 블루투스 기술 등을 사용하여 통신망을 형성하는 무선 네트워크 기술입니다.
