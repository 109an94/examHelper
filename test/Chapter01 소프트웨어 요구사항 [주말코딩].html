2. 구조적/객체지향 다이어그램 구성

  • 클래스
  • 구조적 다이어그램 작성 구성 중 시스템을 구성하는 클래스 사이의 관계를 표현하는 요소는 무엇인지 쓰시오
  • 패키지
  • 구조적 다이어그램 작성 구성 중 클래스나 유스케이스 등 여러 모델 요소를 그룹화하여 패키지를 구성하고, 그 패키지 사이의 관계를 표현하는 요소는 무엇인지 쓰시오
  • 복합체 구조
  • 구조적 다이어그램 작성 구성 중 복합 구조의 클래스와 컴포넌트 내부 구조를 표현하는 요소는 무엇인지 쓰시오
  • 객체
  • 구조적 다이어그램 작성 구성 중 객체 정보를 보여주며 럼바우 객체지향 분석 기법에서 활용되는 요소는 무엇인지 쓰시오
  • 컴포넌트
  • 구조적 다이어그램 작성 구성 중 컴포넌트 구조 사이의 관계를 표현하는 요소는 무엇인지 쓰시오
  • 배치
  • 구조적 다이어그램 작성 구성 중 소프트웨어, 하드웨어, 네트워크를 포함한 실행 시스템의 물리 구조를 표현하는 요소는 무엇인지 쓰시오

6-2. 럼바우 객체지향 분석 (최종 확정 포맷)

  • 객체 모델링 (Object Modeling)
  • 럼바우 OOA에서 시스템 객체를 찾고 객체들 간의 관계를 정의하며, '정보 모델링'이라고도 불리는 모델링 단계를 쓰시오
  • 객체 다이어그램
  • 럼바우 OOA의 객체 모델링 결과물로 나오는 다이어그램을 쓰시오
  • 동적 모델링 (Dynamic Modeling)
  • 럼바우 OOA에서 시간의 흐름에 따른 객체들 사이의 제어 흐름이나 동작 순서 등 동적인 행위를 표현하는 모델링 단계를 쓰시오
  • 상태 다이어그램
  • 럼바우 OOA의 동적 모델링 결과물로 나오는 다이어그램을 쓰시오
  • 기능 모델링 (Functional Modeling)
  • 럼바우 OOA에서 프로세스들의 자료 흐름을 중심으로 처리 과정을 표현하는 모델링 단계를 쓰시오
  • DFD (자료흐름도)
  • 럼바우 OOA의 기능 모델링 결과물로 나오는 다이어그램을 쓰시오
  • 객체 모델링 → 동적 모델링 → 기능 모델링
  • 럼바우 객체지향 분석의 3단계 모델링 순서를 순서대로 쓰시오

3. UML 다이어그램 및 확장

  • 유스케이스
  • UML 다이어그램 유형 중 사용자 관점에서 시스템 행위 표현 (사용자의 요구를 분석하여 시스템의 기능을 명세)
  • 활동
  • UML 다이어그램 유형 중 업무 처리 과정이나 연산이 수행되는 과정을 표현
  • 클래스
  • UML 다이어그램 유형 중 구조 다이어그램과 같으며, 모델링 공간에 객체의 정적인 면을 중심으로 표현
  • 상태 머신
  • UML 다이어그램 유형 중 객체의 생명주기 표현
  • 순차
  • UML 다이어그램 유형 중 시간 흐름에 따른 객체 사이의 상호작용 표현
  • 통신
  • UML 다이어그램 유형 중 객체 사이의 관계를 중심으로 상호작용 표현
  • 상호작용 개요
  • UML 다이어그램 유형 중 상호작용 다이어그램들의 제어 흐름 표현
  • 타이밍
  • UML 다이어그램 유형 중 객체 상태 변화와 시간 제약을 명확하게 표현
  • 스테레오타입, 태그된 값, 제약조건
  • UML 확장 메커니즘 3가지
  • 스테레오타입
  • UML에서 표현하는 기본 기능 외의 추가적인 기능을 표현하는 확장 메커니즘으로, include, extend, interface 등의 종류가 있음

10. 디자인 패턴

  • 생성패턴
  • 객체를 만들 때 생성 목적으로 사용되는 디자인 패턴 그룹
  • 생성 패턴에 속하는 디자인 패턴 5가지
  • 추상팩토리, 빌더, 팩토리메서드, 프로토타입, 싱글턴
  • 소프트웨어 구조를 짤 때 사용되는 디자인 패턴 그룹
  • 구조패턴
  • 어댑터, 브릿지, 컴포지트, 데코레이터, 퍼사드, 플라이웨이트, 프록시
  • 구조 패턴에 속하는 디자인 패턴 7가지
  • 행위패턴
  • 객체들이 어떻게 움직일지 정의할 때 사용되는 디자인 패턴 그룹
  • 스테이트, 스트레티지, 템플릿메서드, 메멘토, 옵서버, 커맨드, 인터프리터, 책임연쇄, 비지터(visitor), 이터레이터, 미디에이터
  • 행위 패턴에 속하는 디자인 패턴 11가지

11. 비용 산정 모형 분류

  • 전문가의 경험적 지식을 통한 문제 해결 및 미래 예측을 위한 하향식 산정 기법
  • 델파이 기법
  • 원시 코드 라인 수의 낙관치, 중간치, 비관치를 측정하여 예측치를 구해 비용을 산정하는 상향식 산정 방법
  • LoC (코드 라인 수)
  • 보헴(Boehm)이 제안한 모형으로 프로그램 규모에 따라 비용을 산정하는 상향식 산정 방법
  • COCOMO
  • 한 사람이 1개월 간 할 수 있는 일의 양을 기준으로 비용을 산정하는 상향식 산정 방법
  • Man Month

12. 소프트웨어 아키텍처 4+1 뷰

  • 유스케이스 뷰
  • 아키텍처를 도출하고 설계하는 작업을 주도하는 뷰

  • 논리 뷰
  • 설계 모델의 추상화, 주요 설계 패키지와 서브 시스템, 클래스 식별을 담당하는 뷰
  • 프로세스 뷰
  • 성능이나 가용성 같은 비기능적 요구사항을 고려하는 뷰

  • 구현 뷰
  • 개발 환경 안에서 정적인 소프트웨어 모듈의 구성을 표현하는 뷰

  • 배포 뷰
  • 물리적인 노드의 구성과 상호연결 관계를 배포 다이어그램으로 표현하는 뷰

    ADR(Active Design Review)
    ARID(Active Review for Intermediate Designs)
