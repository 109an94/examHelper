데이터베이스 실기 플래시카드 (전체 목록)


1. 데이터베이스 구조 및 스키마

  • 스키마
  • 데이터베이스의 구조와 제약 조건에 관한 전반적인 명세를 기술한 것을 의미하는 용어는?
  • 외부 스키마
  • 사용자나 응용 프로그래머가 각 개인의 입장에서 필요로 하는 데이터베이스의 논리적 구조를 정의한 스키마는?
  • 개념 스키마
  • 데이터베이스의 전체적인 논리적 구조로, 조직 전체의 데이터베이스로 하나만 존재하며 사용자들의 데이터를 종합한 것을 의미하는 스키마는?
  • 내부 스키마
  • 물리적 저장장치 입장에서 본 데이터베이스 구조로, 실제로 저장될 레코드의 형식이나 물리적 순서 등을 나타내는 스키마는?

    2. 릴레이션 구성 요소 (기출)
  • 튜플
  • 릴레이션을 구성하는 각각의 행으로, 그 수를 카디널리티 또는 기수라고 하며 레코드와 같은 의미인 것은?
  • 카디널리티 (기수)
  • 릴레이션을 구성하는 각각의 행(튜플)의 수로, 특정 데이터 집합의 유니크한 값의 개수를 나타내는 지표는?
  • 차수 (디그리)
  • 한 릴레이션에 들어있는 애트리뷰트(속성)의 수를 의미하는 것은?
  • 속성 (애트리뷰트)
  • 하나의 릴레이션이 표현하는 개체에 저장하고 싶은 정보의 단위를 뜻하는 것은?
  • 릴레이션 인스턴스 (외연)
  • 어느 한 시점에서 릴레이션의 내용(상태), 즉 저장된 데이터 전체를 의미하는 것은?
  • 도메인
  • 하나의 애트리뷰트가 취할 수 있는 같은 타입의 원자들의 집합을 의미하는 것은?

    3. 관계 대수 및 연산자
  • 합집합 (∪)
  • 두 릴레이션의 튜플의 합집합을 구하는 일반 집합 연산자는?
  • 교집합 (∩)
  • 두 릴레이션의 튜플의 교집합을 구하는 일반 집합 연산자는?
  • 차집합 (–)
  • 두 릴레이션의 튜플의 차집합을 구하는 일반 집합 연산자는?
  • 교차곱 (×)
  • 두 릴레이션의 튜플들의 순서쌍을 구하는 일반 집합 연산자는?
  • 선택 (Select, $\sigma$)
  • 조건을 만족하는 튜플의 부분집합을 구해서 새로운 릴레이션을 만들며, 행(가로)에 해당하는 튜플을 구하는 수평 연산은?
  • 프로젝트 (Project, $\pi$)
  • 속성 리스트에 제시된 속성 값만을 추출해 새로운 릴레이션을 만들며, 열(세로)에 해당하는 속성을 추출하고 중복을 제거하는 수직 연산은?
  • 조인 (Join, $\bowtie$)
  • 공통 속성을 중심으로 두 개 릴레이션을 하나로 합쳐서 새로운 릴레이션을 만드는 연산은?
  • 디비전 (Division, $\div$)
  • R(X)와 S(Y) 두 릴레이션이 있을 때, R의 속성이 S의 속성값을 모두 가진 튜플에서 S가 가진 속성을 제외한 속성만을 구하는 연산은?

    4. 정규형 과정 (도부이결다조)
  • 비정규형 → 제1정규형
  • 도메인이 원자값(원자성)이 되도록 정규화하는 과정은?
  • 제1정규형 → 제2정규형
  • 부분적 함수 종속을 제거하여 정규화하는 과정은?
  • 제2정규형 → 제3정규형
  • 이행적 함수 종속을 제거하여 정규화하는 과정은?
  • 제3정규형 → BCNF
  • 결정자이면서 후보키가 아닌 것을 제거하여 정규화하는 과정은?
  • BCNF → 제4정규형
  • 다치 종속을 제거하여 정규화하는 과정은?
  • 제4정규형 → 제5정규형
  • 조인 종속성을 이용하여 정규화하는 과정은?

    5. 관계형 데이터베이스 제약 조건 (무결성)
  • 개체 무결성
  • 기본 테이블의 기본키를 구성하는 어떤 속성도 Null 값이나 중복값을 가질 수 없다는 규정은?
  • 참조 무결성
  • 외래키 값은 Null이거나 참조 릴레이션의 기본키 값과 동일해야 한다는 규정은?
  • 도메인 무결성
  • 주어진 속성 값이 정의된 도메인에 속한 값이어야 한다는 규정은?
  • 사용자 정의 무결성
  • 속성 값들이 사용자가 정의한 제약 조건에 만족되어야 한다는 규정은?
  • NULL 무결성
  • 릴레이션의 특정 속성 값이 Null이 될 수 없도록 하는 규정은?
  • 관계 무결성
  • 릴레이션의 튜플 삽입 가능 여부 또는 릴레이션 간 튜플 관계의 적절성 여부를 지정한 규정은?

    6. 반정규화
  • 반정규화
  • 시스템의 성능 향상과 개발/운영의 편의성을 높이기 위해 정규화된 데이터 모델을 의도적으로 통합, 중복, 분리하여 정규화 원칙을 위배하는 행위는?

    7. 시스템 카탈로그
  • 시스템 카탈로그 (데이터 사전)
  • DBMS 그 자체에 관련이 있는 다양한 객체에 관한 정보를 포함하는 시스템 데이터베이스로, 좁은 의미로 데이터 사전이라고도 불리는 것은?

    8. 트랜잭션의 특성 (ACID)
  • 원자성 (Atomicity)
  • 트랜잭션의 연산은 데이터베이스에 모두 반영되거나 전혀 반영되지 않아야 하는 트랜잭션의 특성은?
  • 일관성 (Consistency)
  • 트랜잭션이 성공적으로 완료되면 언제나 일관성 있는 데이터베이스 상태로 변환되는 트랜잭션의 특성은?
  • 독립성/격리성 (Isolation)
  • 둘 이상의 트랜잭션이 동시에 실행될 때, 어느 하나의 트랜잭션 실행 중에는 다른 트랜잭션의 연산이 끼어들 수 없도록 하는 트랜잭션의 특성은?
  • 지속성/영속성 (Durability)
  • 성공적으로 완료된 트랜잭션의 결과는 시스템이 고장 나더라도 영구적으로 반영되어야 하는 트랜잭션의 특성은?

    9. 접근 통제 기술
  • 임의 접근 통제 (DAC)
  • 데이터에 접근하는 사용자의 신원에 따라 접근 권한을 부여하는 접근 통제 기술은?
  • 강제 접근 통제 (MAC)
  • 주체와 객체의 등급을 비교하여 접근 권한을 부여하는 접근 통제 기술은?
  • 역할 기반 접근 통제 (RBAC)
  • 사용자의 역할에 따라 접근 권한을 부여하는 접근 통제 기술은?

    10. 파티션
  • 파티션
  • 대용량의 테이블이나 인덱스를 작은 논리적 단위로 나누는 것을 의미하는 용어는?
  • 범위 분할
  • 지정한 열의 기준으로 데이터를 분할하는 파티션 방식은?
  • 해시 분할
  • 해시 함수를 적용한 결과값에 따라 데이터를 분할하는 파티션 방식은?
  • 조합 분할
  • 범위 분할로 분할한 다음 해시 함수를 적용하여 다시 분할하는 방식의 파티션은?

    11. 분산 데이터베이스 목표 (투명성)
  • 위치 투명성
  • 데이터베이스의 실제 위치를 알 필요 없이 논리적인 명칭만으로 액세스할 수 있는 분산 데이터베이스의 목표는?
  • 중복 투명성
  • 동일 데이터가 여러 곳에 중복되어 있더라도 사용자는 마치 하나의 데이터만 존재하는 것처럼 사용하는 분산 데이터베이스의 목표는?
  • 병행 투명성
  • 다수의 트랜잭션들이 동시에 실현되더라도 그 트랜잭션의 결과는 서로 영향을 받지 않는 분산 데이터베이스의 목표는?
  • 장애 투명성
  • 트랜잭션, 네트워크, 컴퓨터, DBMS 장애에도 불구하고 트랜잭션을 정확하게 처리하는 분산 데이터베이스의 목표는?

    12. 클러스터

  • 데이터 액세스의 효율을 향상시키기 위해 동일한 성격의 데이터를 동일한 데이터 블록에 저장하는 물리적인 방법은?
    13. 데이터 모델 구성 요소
  • 구조
  • 논리적으로 표현된 개체 타입들 간의 관계로서 데이터 구조 및 정적 성질을 표현하는 데이터 모델의 구성 요소는?
  • 연산
  • 데이터베이스에 저장된 실제 데이터를 처리하는 작업에 대한 명세로, 데이터베이스를 조작하는 도구인 데이터 모델의 구성 요소는?
  • 제약 조건
  • 데이터베이스에 저장될 수 있는 실제 데이터의 논리적인 제약 조건을 의미하는 데이터 모델의 구성 요소는?

    14. 함수적 종속성
  • 함수적 종속성 (Functional Dependency)
  • 속성 X를 이용하여 속성 Y를 식별할 수 있을 때, '속성 Y가 속성 X에 함수적으로 종속된다'고 말하는 것을 의미하는 용어는?
  • 완전 함수 종속
  • 함수적 종속성의 종류 중 하나는?
  • 부분 함수 종속

    15. SQL 분류
  • DDL (Data Definition Language)
  • SQL 명령어 중 CREATE, ALTER, DROP, RENAME, TRUNCATE 등을 포함하는 분류는?
  • DML (Data Manipulation Language)
  • SQL 명령어 중 SELECT, INSERT, UPDATE, DELETE 등을 포함하는 분류는?
  • DCL (Data Control Language)
  • SQL 명령어 중 GRANT, REVOKE 등을 포함하는 분류는?
  • TCL (Transaction Control Language)
  • SQL 명령어 중 COMMIT, ROLLBACK, SAVEPOINT 등을 포함하는 분류는?
  • GRANT
  • SQL에서 특정 사용자(kim, lee 등)에게 데이터베이스 객체(student 테이블 등)에 대한 접근 권한을 부여하는 명령어를 쓰시오
  • REVOKE
  • SQL에서 특정 사용자에게 부여했던 데이터베이스 객체에 대한 접근 권한을 회수하는 명령어를 쓰시오
