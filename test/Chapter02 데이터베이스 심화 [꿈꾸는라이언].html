Chapter02 데이터베이스 심화 [꿈꾸는 라이언]

  • 이상 (Anomaly) 현상
  • 데이터베이스 설계 시 중복성 및 종속성 문제로 인해 발생하는 현상으로, 삽입 이상, 삭제 이상, 갱신 이상이 있으며 데이터 불일치(정보 모순)를 초래합니다.
  • 삽입 이상 (Insertion Anomaly)
  • 데이터 삽입 시 불필요한 데이터가 함께 삽입되어야 하는 이상 현상입니다.
  • 삭제 이상 (Deletion Anomaly)
  • 튜플 삭제 시 필요한 데이터까지 함께 삭제되는 이상 현상입니다.
  • 갱신 이상 (Update Anomaly)
  • 중복된 데이터 중 일부만 수정되어 데이터가 불일치하는 정보 모순이 발생하는 이상 현상입니다.
  • 정규화 (Normalization)
  • 이상 현상이 발생하지 않도록 중복성과 종속성을 최소화하여 데이터 구조의 안정성을 최대화하고 관리를 용이하게 하는 논리적 설계 단계의 작업입니다.
  • 제 1 정규형 (1NF)
  • 릴레이션의 모든 도메인 (Domain)이 원자 값만으로 되어 있도록 하는 정규형입니다.
  • 제 2 정규형 (2NF)
  • 제 1 정규형을 만족하며, 기본키가 아닌 속성이 기본키에 대해 부분적 함수 종속을 만족하지 않도록 (즉, 완전 함수적 종속을 만족하도록) 하는 정규형입니다.
  • 제 3 정규형 (3NF)
  • 제 2 정규형을 만족하며, 기본키가 아닌 모든 속성이 기본키에 대해 이행적 함수 종속 관계를 만족하지 않도록 (X→Y, Y→Z 일 때 X→Z 관계를 제거) 하는 정규형입니다.
  • BCNF (Boyce-Codd Normal Form)
  • 제 3 정규형을 만족하며, 릴레이션 R에서 모든 결정자 (다른 속성을 결정하는 속성)가 후보키인 정규형입니다.
  • 반정규화 (Denormalization)
  • 정규화 과정 후 시스템 성능 향상 및 운영 편의성을 위해 의도적으로 데이터 중복/통합/분리를 허용하여 정규화 원칙을 위반하는 작업입니다.
  • 완전 함수적 종속 (Full Functional Dependency)
  • 종속자가 기본키 전체에만 종속되며, 기본키를 구성하는 속성의 어떤 부분에도 종속되지 않는 관계입니다.
  • 부분 함수적 종속 (Partial Functional Dependency)
  • 기본키가 여러 속성으로 구성되어 있을 때, 종속자가 기본키를 구성하는 속성 중 일부에만 종속되는 관계입니다.
  • 이행적 함수 종속 (Transitive Functional Dependency)
  • 속성 X, Y, Z가 있을 때, X → Y 이고 Y → Z 이면 X → Z 관계가 성립하는 함수 종속 관계입니다.
  • Select (선택, σ )
  • 관계 대수 순수 관계 연산자 중 하나로, 조건을 만족하는 튜플들의 부분 집합을 추출하는 수평 연산입니다.
  • Project (추출, ㅠ)
  • 관계 대수 순수 관계 연산자 중 하나로, 속성들의 부분 집합을 추출하고 중복을 제거하는 수직 연산입니다.
  • Join (조인, ▷◁ )
  • 관계 대수 순수 관계 연산자 중 하나로, 두 개의 릴레이션을 합쳐 새로운 릴레이션을 형성하는 연산입니다.
  • Division (나누기,÷ )
  • 관계 대수 순수 관계 연산자 중 하나로, 릴레이션 A의 속성이 릴레이션 B의 속성 값을 모두 가진 튜플에서 B가 가진 속성을 제외한 나머지 속성들만 추출하는 연산입니다.
  • Cartesian Product (교차곱, X)
  • 관계 대수 일반 집합 연산자 중 하나로, 두 릴레이션의 가능한 모든 튜플의 집합을 생성하며, 결과 릴레이션의 차수는 더하기, 기수는 곱하기가 됩니다.
  • 트랜잭션 (Transaction)
  • 데이터베이스의 상태를 변환시키는 하나의 논리적 기능을 수행하기 위한 작업 단위입니다.
  • 원자성 (Atomicity)
  • 트랜잭션 연산이 전부 정상적으로 수행되거나 (Commit), 아니면 어떠한 연산도 수행되지 않아야 (Rollback) 한다는 트랜잭션의 ACID 속성입니다.
  • 일관성 (Consistency)
  • 시스템의 고정 요소는 트랜잭션 수행 전과 후가 동일해야 한다는 트랜잭션의 ACID 속성입니다.
  • 독립성 (Isolation)
  • 개별 트랜잭션은 다른 트랜잭션의 간섭 및 영향을 받지 않고 독립적으로 수행되어야 한다는 트랜잭션의 ACID 속성입니다.
  • 영속성 (Durability)
  • 완료된 (Commit 된) 트랜잭션의 결과는 영구적으로 기록되어야 한다는 트랜잭션의 ACID 속성입니다.
  • COMMIT
  • 트랜잭션이 정상적으로 종료된 후 변경된 내용을 데이터베이스에 영구적으로 반영하는 명령어입니다.
  • ROLLBACK
  • 트랜잭션이 비정상적으로 종료된 후 모든 변경 작업을 취소하고 이전 상태로 원복시키는 명령어입니다.
  • REDO
  • 데이터베이스 비정상 종료 시 시작(start)과 완료(commit) 기록이 있는 트랜잭션들의 작업을 재작업하는 (이전 값을 이후 값으로 변경) 회복 연산입니다.
  • UNDO
  • 데이터베이스 비정상 종료 시 시작 기록은 있지만 완료 기록이 없는 트랜잭션들이 작업한 내용을 모두 취소하는 (이후 값을 이전 값으로 변경) 회복 연산입니다.
  • 즉시 갱신 기법 (Immediate Update)
  • 트랜잭션 실행 상태에서 변경되는 내용을 바로 데이터베이스에 적용하고, 변경되는 모든 내용을 로그에 기록하여 장애 시 Redo와 Undo를 모두 수행하는 데이터 회복 기법입니다.
  • 지연 갱신 기법 (Deferred Update)
  • 트랜잭션 수행 후 부분 완료될 때까지 데이터베이스에 적용을 지연시키고, 로그를 토대로 저장하며, 트랜잭션 실패 시 Undo 수행 없이 Redo 만 수행하는 데이터 회복 기법입니다.
  • 로킹 (Locking)
  • 데이터베이스 병행 제어를 위해 트랜잭션이 접근하고자 하는 데이터를 잠가 (Lock) 다른 트랜잭션이 접근하지 못하도록 하는 병행 제어 기법입니다.
  • 로킹 단위 (Locking Granularity)
  • 로킹의 대상이 되는 객체의 크기로, 단위가 작을수록 병행성 (공유도)은 높아지나 로킹 오버헤드가 증가합니다.
  • 낙관적 검증 (Optimistic Concurrency Control)
  • 일단 트랜잭션을 수행하고, 트랜잭션 종료 시에 다른 트랜잭션과의 충돌 여부를 검증하는 병행 제어 기법입니다.
  • 타임 스탬프 기법 (Timestamp Ordering)
  • 트랜잭션에 고유한 시간 정보 (타임 스탬프)를 부여하여, 부여된 시간에 따라 트랜잭션 수행의 순서를 결정하는 병행 제어 기법입니다.
  • 분산 데이터베이스 (Distributed Database)
  • 논리적으로는 하나의 시스템에 존재하나, 물리적으로는 연결된 다수의 컴퓨터에 분산되어 있는 데이터베이스입니다.
  • 위치 투명성 (Location Transparency)
  • 사용자가 데이터베이스의 물리적 위치를 인식할 필요 없이 논리적 이름만으로 접근할 수 있도록 하는 분산 데이터베이스의 목표입니다.
  • 중복 투명성 (Replication Transparency)
  • 사용자가 분산된 데이터의 중복 여부를 인식할 필요 없이 접근할 수 있도록 하는 분산 데이터베이스의 목표입니다.
  • 인덱스 (Index)
  • 데이터베이스 테이블 검색 속도 향상을 위한 저장 위치 자료로, 데이터의 빠른 조회를 위한 위치 정보 (키 값, 포인터)를 포함합니다.
  • 파티셔닝 (Partitioning)
  • 대용량 데이터베이스를 여러 섹션으로 분할하여 데이터 접근 및 조회 속도를 향상시키는 기법입니다.
  • 복구 시간 목표 (RTO, Recovery Time Objective)
  • 서비스 중단 후 복원까지 허용되는 최대 시간으로, 서비스를 사용할 수 없는 최대 허용 시간을 결정합니다.
  • 복구 시점 목표 (RPO, Recovery Point Objective)
  • 마지막 데이터 복구 후 허용되는 최대 데이터 손실량을 결정하는 목표 시간입니다.
