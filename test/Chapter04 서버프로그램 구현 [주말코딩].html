Chapter04 서버프로그램 구현 [주말코딩]

1. 개발 언어 선정 기준

  • 적정성, 효율성, 이식성, 친밀성, 범용성
  • 개발 언어를 선정할 때 고려해야 하는 5가지 주요 기준은 무엇인지 쓰시오

2. 소프트웨어 아키텍처 설계 과정

  • 설계 목표 설정 → 시스템 타입 결정 → 아키텍처 패턴 적용 → 서브 시스템 구체화 → 검토
  • 소프트웨어 아키텍처의 설계 과정을 순서대로 쓰시오

3. 아키텍처 패턴

  • 레이어 패턴
  • 시스템을 계층으로 구분하여 구성하는 고전적인 방법의 아키텍처 패턴은?
  • 클라이언트-서버 패턴
  • 하나의 서버 컴포넌트와 다수의 클라이언트 컴포넌트로 구성되는 아키텍처 패턴은?
  • 파이프-필터 패턴
  • 데이터 스트림 절차의 각 단계를 필터로 캡슐화하여 파이프를 통해 전송하는 아키텍처 패턴은?
  • 모델-뷰-컨트롤러 패턴 (MVC)
  • 서브 시스템을 모델, 뷰, 컨트롤러로 구조화하는 패턴으로, 애플리케이션의 개발 효율성과 유지보수성을 높이는 패턴은?
  • 마스터-슬레이브 패턴, 브로커 패턴, 피어투피어 패턴, 이벤트 버스 패턴, 블랙보드 패턴
  • 주요 4가지 패턴 외에 기타 아키텍처 패턴의 종류를 5가지 이상 쓰시오

4. 객체 지향 분석 방법론

  • 럼바우 방법
  • 모든 소프트웨어 구성 요소를 그래픽 표기법을 이용하여 모델링하는 기법 (객체모델링, 동적모델링, 기능모델링 순으로 이뤄짐)은?
  • 부치 방법
  • 미시적 개발 프로세스와 거시적 개발 프로세스를 모두 사용하는 객체 지향 분석 방법론은?
  • 자콥슨 방법
  • 유스케이스를 강조하여 사용하는 객체 지향 분석 방법론은?
  • Coad & Yourdon 방법
  • E-R 다이어그램을 사용하여 객체의 행위를 모델링하는 객체 지향 분석 방법론은?
  • Wirfs-Brock 방법
  • 분석과 설계 간의 구분이 없으며 고객 명세서를 평가해서 설계 작업까지 연속적으로 수행하는 객체 지향 분석 방법론은?

5. 소프트웨어 아키텍처 설계 기본 원리

  • 모듈화
  • 소프트웨어 성능 향상, 시스템의 수정 및 재사용, 유지 관리 등이 용이하도록 시스템의 기능들을 모듈 단위로 나누는 것은?
  • 추상화
  • 문제의 전체적이고 포괄적인 개념을 설계한 후 차례로 세분화하여 구체화시켜 나가는 것은?
  • 단계적 분해
  • 문제를 상위의 중요 개념으로부터 하위의 개념으로 구체화시키는 분할 기법은?
  • 정보 은닉
  • 한 모듈 내부에 포함된 절차와 자료들의 정보가 감추어져 다른 모듈이 접근하거나 변경하지 못하도록 하는 기법은?

6. 모듈의 독립성 (결합도/응집도)

  • 결합도는 약하게, 응집도는 높게
  • 모듈의 독립성(재사용, 유지 관리 용이)을 높이기 위한 모듈의 결합도와 응집도 조건을 쓰시오
  • 결합도 (Coupling)
  • 모듈 간에 상호 의존하는 정도 또는 두 모듈 사이의 연관 관계를 의미하는 것은?
  • 응집도 (Cohesion)
  • 모듈의 내부 요소들이 서로 연관되어 있는 정도를 의미하는 것은?

7. 결합도 (Coupling) 순서 및 종류

  • 내용 결합도 → 공통 결합도 → 외부 결합도 → 제어 결합도 → 스탬프 결합도 → 자료 결합도
  • 결합도가 강한 순서(내공외제스자)대로 쓰시오
  • 내용 결합도 (Content)
  • 한 모듈이 다른 모듈의 내부 기능 및 내부 자료를 직접 참조하거나 수정하는 결합도는?
  • 공통 결합도 (Common)
  • 공유되는 공통 데이터 영역(전역 변수)을 여러 모듈이 사용할 때의 결합도는?
  • 외부 결합도 (External)
  • 어떤 모듈에서 선언한 데이터 변수를 외부의 다른 모듈에서 참조할 때의 결합도는?
  • 제어 결합도 (Control)
  • 어떤 모듈이 다른 모듈 내부의 논리적인 흐름을 제어하기 위해 제어 요소(flag)를 전달하는 결합도는?
  • 스탬프 결합도 (Stamp)
  • 모듈 간의 인터페이스로 배열이나 오브젝트 포인터 등 동일한 자료 구조가 전달될 때의 결합도는?
  • 자료 결합도 (Data)
  • 모듈 간의 인터페이스가 자료 요소로만 구성되는 (가장 약한) 결합도는?

8. 응집도 (Cohesion) 순서 및 종류

  • 기능적 응집도 → 순차적 응집도 → 교환적 응집도 → 절차적 응집도 → 시간적 응집도 → 논리적 응집도 →

우연적 응집도

  • 응집도가 강한 순서(기순교절시논우)대로 쓰시오
  • 기능적 응집도 (Functional)
  • 모듈 내부 기능 요소들이 단일 문제와 연관되어 수행되는 (가장 강한) 응집도는?
  • 순차적 응집도 (Sequential)
  • 모듈 내 하나의 활동으로 나온 출력 데이터를 그 다음 활동의 입력 데이터로 사용하는 응집도는?
  • 교환적 응집도 (Communication)
  • 동일한 입력과 출력을 사용해 서로 다른 기능을 수행하는 구성 요소가 모인 응집도는?
  • 절차적 응집도 (Procedural)
  • 모듈 안의 구성 요소들이 기능을 순차적으로 수행할 경우의 응집도는?
  • 시간적 응집도 (Temporal)
  • 특정 시간에 처리되는 몇 개의 기능을 모아 하나의 모듈로 작성할 경우의 응집도는?
  • 논리적 응집도 (Logical)
  • 유사한 성격을 갖거나 특정 형태로 분류되는 처리 요소들로 하나의 모듈을 형성하는 응집도는?
  • 우연적 응집도 (Coincidental)
  • 서로 관련 없는 요소들로 하나의 모듈을 구성하는 (가장 약한) 응집도는?

9. 프로세스 간 통신 (IPC)

  • Shared memory
  • 공유 가능한 메모리를 구성하여 다수의 프로세스가 통신하는 IPC 대표 메소드는?
  • Socket
  • 네트워크 소켓을 이용하여 네트워크를 경유하는 프로세스 간에 통신하는 IPC 대표 메소드는?
  • Semaphores
  • 공유 자원에 대한 접근 제어를 통해 통신하는 IPC 대표 메소드는?
  • Pipes & Named Pipe
  • 'Pipe'라고 불리는 선입선출 형태로 구성된 메모리를 여러 프로세스가 공유하여 통신하는 IPC 대표 메소드는?
  • Message Queueing
  • 메시지가 발생하면 이를 전달하는 방식으로 통신하는 IPC 대표 메소드는?

10. 서버 개발 과정 (MVC)

  • DTO/VO (Data Transfer Object/Value Object)
  • 서버 개발 과정에서 데이터 교환을 위해 사용할 객체를 만드는 과정은?
  • SQL
  • 서버 개발 과정에서 데이터 삽입/변경/삭제 등의 작업을 수행할 문을 생성하는 과정은?
  • DAO (Data Access Object)
  • 서버 개발 과정에서 데이터베이스에 접근하고 SQL을 활용하여 데이터를 실제로 조작하는 코드를 구현하는 과정은?
  • Service
  • 서버 개발 과정에서 사용자의 요청에 응답하기 위한 로직을 구현하는 과정은?
  • Controller
  • 서버 개발 과정에서 사용자의 요청에 적절한 서비스를 호출하여 그 결과를 사용자에게 반환하는 코드를 구현하는 과정은?

11. 소프트웨어 재사용을 위한 공통 모듈화

  • 분할과 정복 (Divide & Conquer)
  • 복잡한 문제를 분해해서 모듈 단위로 해결하는 소프트웨어 재사용을 위한 공통 모듈화 기법은?
  • 자료 추상화
  • 함수 내의 자료 구조 표현 내역을 숨기고 인터페이스를 통해 접근하도록 하는 소프트웨어 재사용을 위한 공통 모듈화 기법은?
  • 모듈 독립성
  • 각 모듈의 결합도를 줄이고 응집도를 높여 재사용성을 높이는 공통 모듈화 기법은?
  • 정보 은닉
  • 중요한 정보나 노출 가능성이 큰 모듈을 은닉하는 소프트웨어 재사용을 위한 공통 모듈화 기법은?

12. MVC 패턴 설명

  • MVC (Model-View-Controller)
  • 애플리케이션을 데이터/비즈니스 로직 담당, 사용자 인터페이스 담당, 둘 사이를 연결하는 요소로 분리하여 개발의 효율성과 유지보수성을 높이는 패턴은?
  • Model
  • MVC 패턴에서 데이터와 비즈니스 로직을 담당하며, 데이터를 관리하고 애플리케이션의 상태를 유지하는 요소는?
  • View
  • MVC 패턴에서 사용자 인터페이스(데이터 표시, 버튼, 텍스트 필드 등)를 담당하는 요소는?
  • Controller
  • MVC 패턴에서 사용자의 입력을 처리하고 모델을 업데이트하며 뷰를 업데이트하는, 모델과 뷰 사이를 연결하는 역할을 하는 요소는?
  • 배치 프로그램
  • 일정 스케줄마다 돌아가면서 일괄처리하는 프로그램은?
