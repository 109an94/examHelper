Chapter05 정보보안 [꿈꾸는 라이언]

  • 기밀성 (Confidentiality)
  • 시스템 내 정보 자원을 인가된 사용자에게만 접근을 허용하여, 인가되지 않은 사용자에게는 정보가 노출되지 않도록 하는 정보보안의 3대 요소 중 하나입니다.
  • 무결성 (Integrity)
  • 오직 인가된 사용자만이 시스템 내 정보를 수정할 수 있도록 하여, 정보의 정확성과 완전성을 보장하는 정보보안의 3대 요소 중 하나입니다.
  • 가용성 (Availability)
  • 인가된 사용자는 권한 범위 내에서 언제든지 자원에 접근할 수 있도록 보장하는 정보보안의 3대 요소 중 하나입니다.
  • 인증 (Authentication)
  • 사용자가 네트워크에 접속하기 전에 시스템에서 사용자의 신원을 확인하는 AAA(Triple-A)의 첫 번째 요소입니다.
  • 권한 부여 (Authorization)
  • 신원이 검증된 사용자에게 사용 가능한 접근 권한을 확인하고 허용하는 AAA(Triple-A)의 요소입니다.
  • 계정 관리 (Accounting)
  • 사용자의 자원 사용에 대한 정보(과금, 감사, 보고서 등)를 수집하는 AAA(Triple-A)의 마지막 요소입니다.
  • DES (Data Encryption Standard)
  • IBM에서 개발하여 미국 NBS에서 국가 표준으로 발표되었으며, 블록 크기 64비트와 키 길이 56비트를 사용하고 16라운드로 구성된 양방향 암호화 방식입니다.
  • AES (Advanced Encryption Standard)
  • 미국 표준 기술 연구소(NIST)가 DES를 대체할 목적으로 발표했으며, 키 길이 128/192/256비트를 사용하고 레인달(Rijndael) 기반의 SPN 암호 방식인 양방향 암호화 방식입니다.
  • SEED
  • 한국에서 개발되었으며, 128 및 256비트의 개인키를 사용하는 양방향 암호화 방식입니다.
  • ARIA
  • 국가보안기술연구소(NSRI) 주도로 개발되었으며, 128/192/256비트의 키 길이를 사용하는 양방향 암호화 방식입니다.
  • RC4
  • Ron Rivest가 설계한 스트림 암호화 방식으로, 옥텟 단위 기반으로 동작합니다.
  • RSA (Rivest-Shamir-Adleman)
  • MIT에서 공개키 암호화 방식으로 개발되었으며, 소인수 분해의 어려움을 활용하는 비대칭키 양방향 암호화 방식입니다.
  • Diffie-Hellman (디피-헬만)
  • 두 사용자가 사전에 어떤 비밀 교환 없이도 공통 키를 교환할 수 있도록 하는 최초의 공개키 알고리즘입니다.
  • ECC (Elliptic Curve Cryptography)
  • 타원 곡선 이론에 기반한 공개키 암호 방식입니다.
  • 해시 함수 (Hash Function)
  • 임의의 길이의 입력 데이터를 받아 고정된 길이의 해시값으로 변환하는 단방향 암호화 방식입니다.
  • 중첩법 (Folding Method)
  • 레코드 키를 여러 부분으로 나누고, 나눈 부분의 각 숫자를 더하거나 XOR 한 값을 홈 주소로 사용하는 해시 주소 결정 방식입니다.
  • 제산법 (Division Method)
  • 레코드 키를 해시표의 크기보다 큰 수 중에서 가장 작은 소수로 나눈 나머지를 홈 주소로 삼는 해시 주소 결정 방식입니다.
  • Ping of Death (죽음의 핑)
  • Ping 명령을 전송할 때 허용 범위 이상의 ICMP 패킷을 전송하여 대상 시스템의 네트워크를 마비시키는 서비스 거부 공격입니다.
  • SYN Flooding (SYN 플러딩)
  • ACK 패킷 신호를 고의로 누락한 채 SYN 패킷 신호만 전송하여 서버의 가용 사용자 수를 선점하고 연결 대기 큐를 가득 차게 만들어 다른 사용자의 접근을 차단하는 서비스 거부 공격입니다.
  • Smurfing (스머핑)
  • IP와 ICMP 특성을 악용하여 엄청난 양의 데이터를 한 사이트에 집중적으로 보내 네트워크를 불능 상태로 만드는 서비스 거부 공격입니다.
  • LAND Attack (랜드 어택)
  • 송신 IP 주소와 수신 IP 주소를 모두 타겟 IP 주소로 하여 자기 자신에게 무한히 응답하게 하는 서비스 거부 공격입니다.
  • Phishing (피싱)
  • 악의적 사용자가 지인/유명인을 사칭하여 로그온한 사용자의 계정 정보 및 신용 정보를 탈취하는 공격입니다.
  • Evil twin attack (이블 트윈 공격)
  • 실제 존재하는 것과 동일한 이름의 가짜 무선 WIFI 신호를 송출하여 로그온한 사람들의 계정 정보나 신용 정보를 빼내는 기법입니다.
  • 방화벽 (Firewall)
  • 내부에서 외부로 나가는 패킷은 통과시키고, 외부에서 내부로 유입되는 패킷은 인증된 패킷만 통과시키도록 하여 네트워크 접근을 제어하는 보안 솔루션입니다.
  • 웹 방화벽 (Web Firewall)
  • 일반 방화벽이 탐지하기 어려운 SQL 삽입 공격, XSS 등 웹 기반 공격을 방어할 목적으로 웹 서버에 특화된 방화벽입니다.
  • 침입 탐지 시스템 (IDS, Intrusion Detection System)
  • 시스템의 비정상적인 사용/오용/남용 등을 실시간으로 탐지하고 경고하는 보안 솔루션입니다.
  • 침입 방지 시스템 (IPS, Intrusion Prevention System)
  • 방화벽과 IDS의 기능을 통합하여 비정상적인 트래픽/패킷을 능동적으로 차단하고 격리하는 보안 솔루션입니다.
  • VPN (Virtual Private Network)
  • 공중 네트워크에 암호화 기술을 이용하여 사용자가 마치 자신의 전용 회선을 사용하는 것처럼 안전하게 통신하게 하는 가상 사설 통신망 솔루션입니다.
  • NAC (Network Access Control)
  • 네트워크에 접속하는 내부 PC의 MAC 주소를 IP 관리 시스템에 등록하여 일관된 보안 관리 기능을 제공하는 솔루션입니다.
  • SIEM (Security Information & Event Management)
  • 보안 경고의 실시간 분석을 제공하며, 로그 및 이벤트를 통합 관리하는 빅데이터 기반의 보안 솔루션입니다.
  • Sandbox (샌드박스)
  • 외부에서 유입된 프로그램이 보호된 영역에서 동작하여 시스템이 부정하게 조작되는 것을 막는 보안 형태입니다.
  • FDS (Fraud Detection System)
  • 금융 거래의 이상 거래를 탐지하고 차단하는 시스템입니다.

  • LFSR (Linear Feedback Shift Register)
  • 선형 연산을 통해 다음 상태를 생성하며, 스트림 기반 난수 생성에 활용되는 암호화 방식입니다.
  • IDEA (International Data Encryption Algorithm)
  • 스위스에서 개발되었으며 PES를 개선한 암호키로, 64비트 블록과 128비트 키를 사용하는 양방향 암호화 알고리즘입니다.
  • LEA (Lightweight Encryption Algorithm)
  • 국내 NSRI에서 개발되었으며, 128비트의 데이터 블록을 암호화하는 경량 블록 암호화 알고리즘입니다.
  • Skipjack
  • 미국 NSA가 개발했으며, 64비트 입출력, 80비트의 키, 32라운드를 가지는 블록 암호화 알고리즘입니다.
  • 템퍼프루핑 (Tamper Proofing)
  • 소프트웨어의 위/변조 시 소프트웨어가 오작동하도록 만들어 악용을 방지하며, 해시 함수, 핑거프린트, 워터마킹 등 보안 요소를 생성하여 보호하는 기술입니다.
  • Trust Zone (트러스트 존)
  • 독립적 보안 구역을 따로 두어 중요한 정보를 보호하는 하드웨어 기반의 보안 기술입니다.
  • NAC (Network Access Control)
  • 네트워크에 접속하는 내부 PC의 MAC 주소를 IP 관리 시스템에 등록한 후, 일관된 보안 관리 기능을 제공하는 솔루션입니다.
  • ESM (Enterprise Security Management)
  • 방화벽, IDS/IPS 등 상기 보안 솔루션에서 발생한 로그 기록을 통합하여 관리하고 종합적인 보안 관리 체계를 수립 가능하게 하는 솔루션입니다.
  • DLP (Data Loss Prevention)
  • 내부 정보의 외부 유출을 방지하며, 사내 직원이 사용하는 시스템의 모든 정보를 탐지하고 통제하는 보안 솔루션입니다.
  • Switching Jamming (스위칭 재밍)
  • 위조된 MAC 주소를 지속적으로 보내 스위치 MAC 주소 테이블을 혼란시켜 스위치가 더미 허브처럼 작동하도록 만드는 공격입니다.
  • TearDrop (티어드롭)
  • Fragment number 값을 변형하여 수신 측에서 패킷 조립 시 오류로 인한 과부하로 시스템 다운을 유도하는 서비스 거부 공격입니다.
  • UDP Flooding (UDP 플러딩)
  • 다량의 UDP 패킷을 전송하여 네트워크 자원을 고갈시키고 시스템 에러를 유발하는 서비스 거부 공격입니다.
  • Ping Flooding (핑 플러딩)
  • 과도한 ICMP 메시지 전송으로 인한 응답 과다로 시스템에 에러를 유발하는 서비스 거부 공격입니다.
  • 서비스 거부 공격 (DOS, Denial of Service)
  • 대량의 데이터를 한 곳의 서버에 집중적으로 전송하여 표적이 되는 서버의 정상적인 기능을 방해하는 공격 유형입니다.
  • 분산 서비스 거부 공격 (DDOS, Distributed Denial of Service)
  • 여러 대의 장비 (Trinoo, TFN, Stacheldraht 등)에서 한 곳의 서버에 분산하여 서비스 공격을 수행하는 공격 유형입니다.
