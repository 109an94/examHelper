Chapter04 네트워크 [꿈꾸는 라이언]

OSI 7계층 및 TCP/IP 4계층

  • 응용 계층 (Application Layer)
  • 응용 프로그램이 OSI 환경에 접속 가능한 서비스를 제공하는 계층이며, TCP/IP의 응용 계층에 해당합니다 (Ex. Telnet, HTTP, DNS).
  • 표현 계층 (Presentation Layer)
  • 응용 계층과 세션 계층 간의 코드/데이터 변환, 데이터 암호화/압축 등 데이터의 표현 방식을 정의하는 계층입니다.
  • 세션 계층 (Session Layer)
  • 컴퓨터 간의 세션/연결을 생성, 유지, 종료하여 적절한 통신 상태를 유지하는 계층입니다.
  • 전송 계층 (Transport Layer)
  • 종단(END-to-END) 간 투명한 데이터 전송/전달을 담당하며, 주소 설정, 다중화, 오류 제어, 흐름 제어 등을 수행합니다. TCP/IP의 전송 계층에 해당합니다.
  • 네트워크 계층 (Network Layer)
  • 개방 시스템 간 네트워크 연결을 관리하고, 경로 설정 (Routing), 트래픽 제어, 패킷 정보 전송 등을 담당합니다. TCP/IP의 인터넷 계층에 해당합니다.
  • 데이터 링크 계층 (Data Link Layer)
  • 송/수신 간 속도 차이 해결을 위한 흐름 제어 기능, 프레임 동기화, 오류 제어, 물리적 주소 (MAC) 결정 등을 수행하며, 노드 간 프레임 전송을 담당하는 계층입니다. TCP/IP의 네트워크 액세스 계층에 해당합니다.
  • 물리 계층 (Physical Layer)
  • 전송에 필요한 기계적, 전기적, 기능적 특성을 정의하고, 데이터를 0과 1의 전기적 신호로 변환하는 최하위 계층입니다. TCP/IP의 네트워크 액세스 계층에 해당합니다.

HDLC 프로토콜

  • HDLC (High Level Data Link Control)
  • 대표적인 데이터 링크 프로토콜로, 비트 동기 방식을 사용하며 데이터 전송 제어, 흐름 제어, 오류 제어 등의 기능을 수행합니다.
  • 정규 응답 모드 (NRM, Normal Response Mode)
  • HDLC 전송 모드 중 **주국과 종국의 관계 (불균형)**를 가지며, 종국이 데이터 전송 시 주국의 허락이 필요한 모드입니다.
  • 비동기 균형 모드 (ABM, Asynchronous Balanced Mode)
  • HDLC 전송 모드 중 **대등한 혼합국 간의 관계 (균형)**를 가지며, 양쪽에서 명령과 응답 모두 전송 가능한 모드입니다.
  • 정보 프레임 (Information Frame)
  • HDLC 프레임 종류 중 제어부가 0으로 시작하며, 사용자 데이터와 일부 제어 정보를 전달하는 프레임입니다. **송신용 순서 번호 (Seq)**와 **응답용 순서 번호 (Next)**를 포함합니다.
  • 감독 프레임 (Supervisory Frame)
  • HDLC 프레임 종류 중 제어부가 10으로 시작하며, 확인 응답, 흐름 제어, 오류 제어 용도로 사용되고 Next 번호만 존재하는 프레임입니다.

네트워크 및 전송 계층 프로토콜

  • IP (Internet Protocol)
  • 비연결형 데이터그램 방식으로 전송할 데이터에 주소 지정 및 경로를 설정하며, 헤더 체크섬만 제공하고 신뢰성을 보장하지 않는 네트워크 계층 프로토콜입니다.
  • ICMP (Internet Control Message Protocol)
  • IP와 함께 통신 간 오류 처리와 전송 경로 변경 등 제어 메시지를 관리하는 네트워크 계층 프로토콜입니다.
  • ARP (Address Resolution Protocol)
  • 호스트의 IP 주소를 네트워크 접속 장비의 **물리적 주소 (MAC 주소)**로 변환하는 네트워크 계층 프로토콜입니다.
  • RARP (Reverse Address Resolution Protocol)
  • **물리적 주소 (MAC 주소)**를 IP 주소로 변환하는 기능을 수행하는 네트워크 계층 프로토콜입니다.
  • NAT (Network Address Translation)
  • 사설 네트워크 IP 주소를 공인 IP 주소로 변환하여, 하나의 공인 IP 주소에서 다량의 가상 사설 IP 주소 할당 및 연결을 가능하게 하는 기술입니다.
  • TCP (Transmission Control Protocol)
  • 양방향 서비스를 제공하며, 순서 제어, 오류 제어, 흐름 제어 기능으로 높은 신뢰성을 보장하는 전송 계층 프로토콜입니다.
  • UDP (User Datagram Protocol)
  • 비연결형 서비스를 제공하며, TCP 대비 단순한 헤더 구조로 오버헤드가 적고 전송 속도가 빠르며 신뢰성보다는 속도가 중요한 실시간 전송에 유리한 전송 계층 프로토콜입니다.
  • RTCP (Real-time Control Protocol)
  • RTP 패킷의 전송 품질을 제어하며, 세션에 참여한 각 참여자들에게 주기적으로 제어 정보를 전송하는 전송 계층 프로토콜입니다.

응용 계층 프로토콜

  • Telnet
  • 다른 컴퓨터에 접속하여 원격 서비스를 제공하는 프로토콜로, 가상의 터미널 기능을 수행하며 기본 포트 23번을 사용합니다.
  • FTP (File Transfer Protocol)
  • 원격 파일 전송을 위한 프로토콜로, 컴퓨터와 인터넷 간 파일 전송 서비스를 제공하며 기본 포트 21번을 사용합니다.
  • HTTP (Hyper Text Transfer Protocol)
  • WWW 내 HTML 문서를 송수신하기 위한 프로토콜로, GET과 POST 메서드를 통해 메시지를 주고받으며 기본 포트 80번을 사용합니다.
  • SMTP (Simple Mail Transfer Protocol)
  • 전자 우편 교환 서비스를 위한 프로토콜로, 기본 포트 25번을 사용합니다.
  • DNS (Domain Name System)
  • 도메인 네임을 IP 주소로 매핑(변환)하는 시스템입니다.
  • DHCP (Dynamic Host Configuration Protocol)
  • IP 주소 및 TCP/IP 프로토콜의 기본 설정을 클라이언트에게 자동으로 제공하는 프로토콜입니다.

TCP/IP 헤더 구조 및 라우팅

  • TTL (Time to live)
  • IP 헤더 구조의 필드 중 하나로, 패킷이 네트워크를 통과할 수 있는 남은 라우터의 최대 개수를 나타냅니다.
  • Sequence Number (일련 번호)
  • TCP 헤더 구조의 필드 중 하나로, 송신자가 지정한 순서 번호를 전달하여 원활한 흐름 제어를 목적으로 합니다.
  • Acknowledgment Number (확인 응답 번호)
  • TCP 헤더 구조의 필드 중 하나로, 수신자로부터 송신자에게 다음번 데이터가 몇 번째인지 전달하는 번호입니다.
  • SYN 플래그 비트
  • TCP 플래그 중 하나로, **연결 동기화 (세션 연결 시작)**를 요청하는 비트입니다.
  • FIN 플래그 비트
  • TCP 플래그 중 하나로, 연결 종료를 요청하는 비트입니다.
  • 가상 회선 방식 (Virtual Circuit)
  • 패킷 교환 방식 중 전송 완료 전 논리적 통신 경로를 미리 설정하고, 별도의 가상 회선으로 송/수신 간 데이터를 전달하는 방식입니다.
  • 데이터그램 방식 (Datagram)
  • 패킷 교환 방식 중 연결 경로 사전 설정 없이 각 패킷을 순서에 무관하게 독립적으로 전송하는 방식입니다.
  • RIP (Routing Information Protocol)
  • 거리 벡터 라우팅 프로토콜로, 최대 15홉을 지원하며 Bellman-Ford 알고리즘을 사용하여 경로를 라우팅하는 IGP (Interior Gateway Protocol)입니다.
  • OSPF (Open Shortest Path First Protocol)
  • 대규모 네트워크에 널리 사용되며, 실시간 노드 간 거리 및 링크 상태를 반영하여 Dijkstra 알고리즘을 사용하는 IGP (Interior Gateway Protocol)입니다.
  • BGP (Border Gateway Protocol)
  • 서로 다른 AS 간 라우팅 정보를 교환하는 **EGP (Exterior Gateway Protocol)**로, Path Vector 기반 라우팅을 수행합니다.
  • L2TP (Layer 2 Tunneling Protocol)
  • VPN 관련 프로토콜 중 L2FP와 PPTP의 기능이 결합되었으며, 인터넷 내 두 지점 간 가상의 터널을 만들어 통신을 안전하게 전송하는 프로토콜입니다.
