class 인공지능

  • 인공지능 기술
  • 인간의 학습능력, 추론능력, 지각능력을 인공적으로 구현하려는 과학 기술
  • 알고리즘
  • 컴퓨터 과학에서 어떠한 문제를 풀기 위한 일련의 절차와 방법을 공식화한 형태로 표현
  • 튜링 테스트
  • 최초의 컴퓨터, 인간과 같은 지능을 가지는 기계 를 판별하는 목적
  • 다트머스 회의
  • 에서 존 매카시가 최초록 생각하는 기계 → 인공지능 사용
  • 백트래킹ㅇ탐색 추론
  • 문제의 정답을 단계별로 찾아가다 실패할 경우 이전 단계에서 다른 해를 찾아나가는 최적화 기법 :
  • FSD 완전 자율주행 - 오토 파일럿, FDS 이상 거래 탐지 시스템, DALL-E 이미지 생성 모델, 코파일럿 등,
  • 인공지능 적용 기술
  • ChatGPT 의 대규모 언어 모델
  • 방대한 양의 텍스트 데이터를 학습하여 사람들이 언어를 사용하는 방식을 흉내
  • 딥페이크 기술
  • 딥러닝을 이용한 가짜 이미지나 동영상을 생성하는 인공지능 기술

  • 모라벡의 역설
  • 인간에게 단순해 보이는 낮은 수준의 기술을 구현하려면 많은 양의 연산 자원
  • 약한 인공지능
  • : 제한적으로 좁은 영역에 대해 적용 (하나의 작업에 특화 , 알파고)
  • 강한 인공지능
  • : 인간처럼 지속적으로 배우고, 여러 도메인 지식을 연결, 다기능적 요소
  • 초 인공지능
  • : 다면적 지능을 넘어 인간보다 빠르고 정확한 의사 결정
  • 지도학습
  • 치와와 VS 머핀 구별
  • 1차 산업 혁명
  • 증기기관 기반의 기계화 혁명 - 대량생산
  • 2차 산업 혁명
  • : 석유와 전기 에너지 기반의 대량생산
  • 3차 산업 혁명
  • : 지식 정보 혁명 - 서비스기반 혁명
  • 4차 산업 혁명
  • : 지능 정보 기술을 산업과 서비스에 결합
  • 퍼셉트론의 출현
  • 뇌세포와 유사한 인공 신경망을 어떻게 학습
  • 머신러닝
  • : 데이터 기반으로 학습을 수행 , 규칙을 수정
  • 딥러닝
  • : 학습과 오류의 수정을 통해 품질을 스스로 개선 → 스스로 시행착오를 겪음
  • 불쾌한 골짜기
  • : 인간과 가까워질 수록 호감도가 급격히 떨어짐
  • 빅데이터
  • : 컴퓨터나 스마트폰을 통해서 생성된 방대한 데이터 를 학습데이터로 사용
  • 데이터 분석 :
  • 가공하는 작업 → 의미 있는 정보 얻는 일

  • 확증 편향(선택적 인지
  • ) : 원하는 정보만 선택적으로 모으거나 어떤 것을 설명하거나 주장할 떄 편향된 방법
  • 데이터 표본 편향
  • : 자료 수집 단계의 문제로 데이터 샘플링 편향
  • 인간 편향 , 숨겨진 편향, 롱테일 편향, 고의적 편향
  • 그외 편향
  • 사람들에게 허구의 말과 행동을 하는 것을 보여줌
  • 딥페이크 의 윤리 문제
  • 아실로마 인공지능 원칙
  • : 생산성 증대와 기술 혁신에 의한 경제 성장으로 인간의 삶이 윤택해졌다고 강조함 (낙관+비관)
  • 모럴 머신 (자율주행차의 윤리)
  • 인공지능이 윤리적 결정에 대한 사회적 인식을 수집하는 작업? , 인간의 윤리적 판단을 통해 사회적 합의
  • 공리주의
  • : 최대 다수의 최대 행복
  • 저급 언어
  • : 기계가 바로 이해하여 수행 → 0과 1로 구성 - 수행속도가 빠름
  • 고급 언어
  • : 사람이 쉽게 이해할 수 있는 언어, 컴파일러와 인터프리터와 같은 해석과 실행을 하는 별도의 프로그램
  • 컴파일러 방식
  • : 기계어로 해석하여 실행시 기계어를 메모리에서 실행 (자바) / 전체 단위 해석 - 후 실행
  • 인터프리터 방식
  • : 즉시 해석하고 기계에서 실행 (파이썬) / 실시간 디버깅 가능 / 줄 단위로 실행
  • 명시적 프로그램
  • : 규칙을 명시하고 이 규칙에 따라 수행
  • 기호주의적 시도
  • : 인간의 사고를 완벽하게 표현할 수 있다면 인공지능을 구현할 수 있지 않을까?
  • 규칙 기반 모델
  • : 인간의 지능과 유사한 일을 할 수 있을 것이다.
  • 전문가 시스템
  • : 고도의 현실적 문제를 전문가와 동등한 레벨에서 해결하는 지적 인공지능 시스템
  • 머신러닝 :
  • 데이터 학습을 통해 규칙을 찾는 프로그램 (자동)
  • 명시적 프로그램 :
  • 의 한계 사람이 세부적인 규칙을 일일히 알려줘야 함 (수동)
  • ASCII
  • : 8bit → 1 byte 는 영문 한 글자와 특수 기호를 표현하는 단위
  • 유니코드
  • : 2바이트를 묶어서 하나의 문자로 사용 (2byte가 국제 표준 )
  • 빅데이터
  • : DB관리 도구의 능력을 넘어 정형-비정형 데이터 집합으로 부터 가치를 추출, 결과를 분석
  • : 양 VOLUME, 속도 VELOCITY,종류 VARIETY
  • ㅇV3
  • 비정형 데이터
  • : 정해진 규칙이 없어서 값의 의미를 파악하기 힘든 데이터
  • 정형 데이터
  • : 행&열로 표현된 정제된 데이터, 의미 파악이 쉽고 규칙적인 값으로 이루어짐
  • 사물인터넷
  • : IOT 사물의 센서와 통신 기능 등
  • 온톨로지
  • : 인간이 보고 듣고 느끼고 생각하는 것에 대해 컴퓨터에서 처리할 수 있는 형태로 표현한 모델
  • 프로그램 내장 방식
  • : 존 폰 노이만에 의해 고안 / 주기억 장치가 (메모리) 데이터 저장 → 중앙처리장치 (CPU) 가 처리하는 방식
  • CPU
  • : 1~8개 가량 적은 코어 / 직렬 처리 방식 / 컴퓨터의 연산과 제어를 담당 / 빠른 병렬 연산이 어려움
  • GPU
  • 그래픽 처리를 담당, 인공지능을 위한 병렬처리 / 병렬 처리 방식 / 병렬 연산 이외의 복잡한 처리가 어려움 / 느림
  • GPGPU와 형용모순
  • (안나올듯) : GP(범용) ←형용모순→ GPU(그래픽 처리 전용) / 그래픽용으로 만들어진건데 GP 범용 기능을 추가하여 CPU대신 다른 범용 하드웨어로도 씀 → 딥러닝시스템에 최적화
  • 클라우드 컴퓨팅
  • : 컴퓨터 자원을 인터넷 어딘가에서 가져와 사용하는 것
  • 데이터 센터 :
  • 서버 컴퓨터와 회선을 제공하는 건물이나 집적화 된 시설 → 안정적 관리를 위해
  • 지능형 로봇 구성 :
  • 매니퓰레이터 (기구부), 엔추에이터(근육), 센서 (오감)
