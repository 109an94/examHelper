Chapter03 운영체제01 [꿈꾸는 라이언]

운영체제 종류 및 구성

  • Windows (윈도우)
  • 마이크로소프트사에서 개발한 운영체제입니다.
  • UNIX (유닉스)
  • 미국 AT&T 벨 연구소에서 개발한 운영체제로, 커널과 쉘로 구성되어 있습니다.
  • Linux (리눅스)
  • 리누스 토발즈가 유닉스를 기반으로 개발한 운영체제입니다.
  • Android (안드로이드)
  • 구글이 리눅스 커널 기반으로 개발한 개방형 모바일 운영체제입니다.
  • 커널 (Kernel)
  • 유닉스 구성 요소 중 하드웨어를 보호하고 프로그램 및 하드웨어 간 인터페이스 역할을 하며, 프로세스 관리, 기억장치 관리, 파일 관리 등 OS의 핵심 기능을 수행하는 부분입니다.
  • 쉘 (Shell)
  • 유닉스 구성 요소 중 사용자의 명령을 인식/해석한 후 커널로 전달하여 명령을 수행하게 하는, 시스템과 사용자 간의 인터페이스 역할을 하는 부분입니다.

주기억장치 및 기타 메모리

  • RAM (Random Access Memory)
  • 휘발성 메모리로, 전원이 꺼지면 저장된 데이터가 사라지며, 실행 중인 프로그램을 저장하는 데 사용되는 주기억장치입니다.
  • SRAM (Static RAM)
  • 빠른 속도와 낮은 집적도를 가지며, 재충전이 불필요하여 주로 캐시 메모리로 사용되는 휘발성 메모리입니다.
  • DRAM (Dynamic RAM)
  • 느린 속도와 높은 집적도를 가지며, 재충전이 필요하고 주로 주기억장치로 사용되는 휘발성 메모리입니다.
  • ROM (Read Only Memory)
  • 전력 공급이 없어도 내용이 사라지지 않는 비휘발성 메모리이며, 하드웨어와 소프트웨어의 중간인 **펌웨어(Firmware)**로 활용됩니다.
  • EPROM (Erasable Programmable ROM)
  • 자외선 신호를 이용하여 저장된 내용을 지우고 다시 기록할 수 있는 ROM입니다.
  • EEPROM (Electrically Erasable Programmable ROM
  • 전기 신호를 이용하여 저장된 내용을 지우고 다시 기록할 수 있는 ROM이며, 플래시 메모리로 활용되어 BIOS ROM 업데이트 등에 사용됩니다.
  • 레지스터 (Register)
  • CPU 내부에서 처리할 명령어나 연산의 중간값 등을 일시적으로 저장하는 휘발성 메모리로, 메모리 중 가장 빠른 속도를 가집니다.
  • 캐시 메모리 (Cache Memory)
  • SRAM을 이용하여 CPU(빠름)와 주기억장치(느림) 사이의 속도 차이를 해결하는 메모리이며, 적중률이 높을수록 시스템 처리 속도가 증가합니다.
  • 가상 메모리 (Virtual Memory)
  • 보조 기억장치의 일부를 주기억장치처럼 사용하여 기억 용량을 확대하고, 큰 프로그램도 실행 가능하게 하는 메모리 관리 기술입니다.
  • 버퍼 메모리 (Buffer Memory)
  • 데이터를 일시적으로 저장해 두 장치 간의 데이터 전송 속도 차이를 해결하는 데 사용되는 메모리입니다.
  • 연상(연관) 메모리 (Associative Memory)
  • 주소가 아닌 저장된 데이터의 내용의 일부를 이용하여 기억장치에 접근하고 데이터를 확인하는 메모리입니다.

기억장치 관리 전략

  • 요구 반입 (Demand Fetch)
  • 특정 프로그램이나 데이터의 참조를 '요구'할 때 해당 데이터를 주기억장치에 적재하는 반입 전략입니다.
  • 예상 반입 (Anticipatory Fetch)
  • 미래의 참조가 '예상'되는 데이터를 미리 주기억장치에 적재하는 반입 전략입니다.
  • 최초 적합 (First Fit)
  • 사용 가능한 분할 영역 중에서 크기가 충분한 '첫 번째' 분할 영역에 데이터를 배치하는 배치 전략입니다.
  • 최적 적합 (Best Fit)
  • 사용 가능한 분할 영역 중에서 데이터 크기와 가장 가까워 단편화를 '최소화'하는 분할 영역에 데이터를 배치하는 배치 전략입니다.
  • 최악 적합 (Worst Fit)
  • 사용 가능한 분할 영역 중에서 가장 큰 영역에 데이터를 배치하여 **단편화를 '최대화'**하는 배치 전략입니다.

가상 기억장치 관리 기법

  • 페이징 기법 (Paging)
  • 가상 기억장치의 프로그램 (페이지)과 주기억장치의 영역 (페이지 프레임)을 동일한 크기로 분할하여 배치 및 실행하는 기법입니다. (내부 단편화 발생 가능)
  • 세그먼테이션 기법 (Segmentation)
  • 가상 기억장치의 프로그램 (세그먼트)을 다양한 크기의 논리적 단위로 분할하여 주기억장치에 배치 및 실행하는 기법입니다. (외부 단편화 발생 가능)
  • OPT (Optimal Replacement)
  • 앞으로 가장 오랫동안 사용하지 않을 페이지를 교체하는 페이지 교체 알고리즘으로, 페이지 부재 횟수가 가장 적게 발생하는 최적의 알고리즘입니다.
  • FIFO (First-In, First-Out)
  • 주기억장치에 가장 먼저 들어와 가장 오래 있었던 페이지를 교체하는 페이지 교체 알고리즘입니다.
  • LRU (Least Recently Used)
  • 최근에 가장 오랫동안 사용하지 않은 (가장 오래전에 사용된) 페이지를 교체하는 페이지 교체 알고리즘입니다.
  • LFU (Least Frequently Used)
  • 사용 빈도가 가장 적은 페이지를 교체하는 페이지 교체 알고리즘입니다.
