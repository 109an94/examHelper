Chapter02 알고리즘 및 정렬 [꿈꾸는 라이언]

  • 선형 검색 (Linear Search / Sequential Search)
  • 데이터 집합에서 원하는 결과값을 하나씩 순서대로 비교하며 찾아내는 검색 방법입니다.
  • 이진 검색 (Binary Search)
  • 검색을 수행하기 전에 데이터 집합이 정렬되어 있어야 하며, 중간 값을 기준으로 검색 범위를 절반씩 줄여가며 원하는 값을 찾아내는 검색 방법입니다.
  • O(1) 시간 복잡도
  • 해시 함수 등에서 나타나며, 자료 크기에 무관하게 일정한 속도를 가지는 상수형 복잡도입니다.
  • O(log₂N) 시간 복잡도
  • 이진 탐색 등에서 나타나며, 자료 크기가 커질수록 처리 시간이 로그 함수처럼 느리게 증가하는 로그형 복잡도입니다.
  • O(n) 시간 복잡도
  • 순차 탐색 등에서 나타나며, 입력 자료를 하나씩 처리하여 처리 시간이 자료 크기에 정비례하는 선형 복잡도입니다.
  • O(N log₂N) 시간 복잡도
  • 힙 정렬, 합병(병합) 정렬 등에서 나타나며, 선형 로그형 복잡도를 가집니다.
  • O(N²) 시간 복잡도
  • 선택 정렬, 버블 정렬, 삽입 정렬 등에서 나타나며, 자료 크기가 커질수록 처리 시간이 제곱하여 증가하는 복잡도입니다.
  • Divide and Conquer (분할 정복)
  • 큰 문제를 최소 단위로 나누어 해결하고, 각각의 해답을 다시 병합하여 최종 해답을 얻는 알고리즘 설계 기법입니다.
  • Dynamic Programming (동적 계획법)
  • 문제를 더 작은 문제의 연장선으로 여기고, 과거에 구한 해 (중간 결과)를 재활용하여 전체 문제의 해를 구하는 알고리즘 설계 기법입니다.
  • Greedy (탐욕 알고리즘)
  • 현 시점에서 가장 최적인 방법을 선택하여 최종 해답에 도달하려는 알고리즘 설계 기법입니다.
  • Backtracking (백트래킹)
  • 해를 찾는 과정에서 모든 조합을 시도해 보되, 해가 될 가능성이 없는 경우 문제의 부모 노드로 되돌아가 (Backtrack) 다른 자손 노드를 검색하는 알고리즘 설계 기법입니다.
  • 삽입 정렬 (Insertion Sort)
  • 이미 정렬된 파일에 새로운 레코드를 순서에 맞게 삽입하여 정렬하며, 두 번째 숫자부터 앞서 정렬된 숫자들 사이에 크기 순에 맞게 재정렬하는 과정을 반복하는 정렬 알고리즘입니다.
  • 선택 정렬 (Selection Sort)
  • N개의 레코드 중 최소값을 찾아 첫 번째에 배치하고, 나머지 N-1개 레코드 중 최소값을 찾아 두 번째에 배치하는 과정을 반복하여 정렬하는 알고리즘입니다.
  • 버블 정렬 (Bubble Sort)
  • 인접한 두 개의 레코드 키 값을 비교한 후 크기에 따라 위치를 교환하며, 1 Pass 마다 정렬되는 숫자들 중 가장 큰 숫자를 오른쪽으로 밀어서 배치하는 정렬 알고리즘입니다.
  • 퀵 정렬 (Quick Sort)
  • 자료 이동을 최소화하며 하나의 파일을 부분적으로 나눠가면서 (Divide) 정렬하는 알고리즘입니다.
  • 합병 정렬 (Merge Sort)
  • 이미 정렬된 2개 이상의 파일을 하나의 파일로 합병하여 정렬하는 알고리즘입니다 (2-way 합병 등).
  • 쉘 정렬 (Shell Sort)
  • 입력 파일을 매개변수 값으로 서브파일을 구성한 후, 각 서브파일을 삽입 정렬 방식으로 순서 배열하는 과정을 반복하는 정렬 알고리즘입니다.
  • 힙 정렬 (Heap Sort)
  • 완전 이진 트리를 이용하여 최대값이나 최소값이 루트 노드에 오도록 구조를 만든 후 정렬을 수행하는 알고리즘입니다.
  • 해시 함수 (Hash Function)
  • 자료 크기에 무관하게 O(1)의 일정한 속도로 실행되며, 주어진 데이터를 고정된 길이의 값으로 변환하여 저장 위치를 빠르게 찾아낼 수 있도록 하는 알고리즘입니다.
  • 이진 탐색 (Binary Search)
  • O(log₂N)의 로그형 복잡도를 가지며, 검색 수행 전에 데이터 집합이 정렬되어 있어야만 중간 값을 기준으로 검색 범위를 절반씩 줄여가며 원하는 값을 찾아내는 효율적인 검색 알고리즘입니다.
  • 순차 탐색 (Sequential Search / Linear Search)
  • O(n)의 선형 복잡도를 가지며, 입력 자료를 하나씩 순서대로 비교하며 원하는 결과값을 찾아내는 가장 기본적인 검색 알고리즘입니다.
  • 힙/합병(병합) 정렬 (Heap/Merge Sort)
  • O(N log₂N)의 선형 로그형 복잡도를 가지며, 합병 정렬은 정렬된 여러 파일을 하나로 합치거나, 힙 정렬은 완전 이진 트리를 이용하는 비교적 효율성이 좋은 정렬 알고리즘입니다.
  • 선택/버블/삽입 정렬 (Selection/Bubble/Insertion Sort)
  • O(N²)의 제곱 복잡도를 가지며, 자료 이동이 많아 N이 큰 경우 효율성이 낮은 정렬 알고리즘들입니다. (예: 버블 정렬은 인접한 두 값을 비교하여 교환하는 방식입니다.)
